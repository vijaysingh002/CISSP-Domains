Security and Risk Management

This domain focuses on identifying and managing risks to an organization’s information systems. It includes developing security policies, conducting risk assessments, and ensuring compliance with laws and regulations.

Example: Implementing a policy that requires encryption for all sensitive customer data to mitigate the risk of a data breach.

Asset Security

Asset Security involves the proper handling and protection of organizational assets, including both physical and digital data. This domain covers the classification, storage, retention, and destruction of data throughout its lifecycle. Ensuring confidentiality, integrity, and availability of data is key.

Example: Implementing a data classification policy that ensures sensitive data is handled securely and only accessible to authorized personnel. Securing backups in off-site locations for disaster recovery purposes.

Communication and Network Security

This domain focuses on securing network infrastructures, communication channels, and protecting data transmitted across these networks. It covers designing secure network architectures, managing external connections, and applying encryption to ensure data safety during transmission.

Example: Using a Virtual Private Network (VPN) to securely connect remote employees to internal corporate systems, ensuring encrypted communication over the internet.

Security Architecture and Engineering

Security Architecture and Engineering focus on designing secure systems and infrastructures that incorporate security controls to protect hardware, software, and networks. This includes applying security principles like defense-in-depth, least privilege, and zero trust models.

Example: Implementing multi-layered security controls like firewalls, intrusion detection systems, and encryption to safeguard critical systems and data from cyber threats.

Identity and Access Management (IAM)

Identity and Access Management ensures that only authorized individuals have access to specific resources within the organization. This includes processes for identification, authentication, authorization, and accountability to manage user identities and permissions securely.

Example: Implementing multi-factor authentication (MFA) for accessing sensitive systems, requiring users to verify their identity through multiple forms of authentication.

Security Operations

Security Operations involve continuous monitoring, managing security incidents, and responding to breaches. This domain ensures that security controls are effectively enforced, incidents are handled quickly, and post-incident investigations are carried out to improve future security posture.

Example: Deploying a Security Information and Event Management (SIEM) system to detect and respond to unusual activities that could indicate a cyber attack.

Security Assessment and Testing

This domain involves evaluating and testing the effectiveness of security measures. It includes security audits, vulnerability assessments, and penetration testing to identify weaknesses in systems and ensure security controls are functioning as intended.

Example: Conducting regular penetration testing to identify vulnerabilities in the organization’s web applications and infrastructure, followed by remediation of those vulnerabilities.

Software Development Security

Software Development Security ensures that security is integrated throughout the software development lifecycle (SDLC). It focuses on secure coding practices, threat modeling, and regular security testing to prevent vulnerabilities in software applications.

Example: Conducting security code reviews during the development phase to ensure that common vulnerabilities such as SQL injection and cross-site scripting (XSS) are mitigated.
